# 🏥 Community Health Intelligence Assistant

> **AI for Better Living and Smarter Communities**

## 🎯 Judge-Facing Pitch

The **Community Health Intelligence Assistant** transforms disconnected medical report PDFs into actionable health intelligence at two levels: it helps **individual patients** understand their lab results through AI-powered plain-language explanations, while simultaneously enabling **community health workers, clinics, and public health planners** to surface population-level trends — like rising HbA1c rates or regional anemia spikes — from aggregated, anonymized data. Built on a **Retrieval-Augmented Generation (RAG)** pipeline with automatic **anomaly detection** and a structured analytics layer, the platform embodies the hackathon's vision of a **decision intelligence platform** that predicts outcomes and helps communities make better, data-driven health decisions. Every AI response is grounded in source evidence, with full transparency and medical disclaimers — delivering **responsible AI** that improves everyday life.

---

## 📋 Problem Statement

### Individual Level
Medical reports contain complex terminology, abbreviations, and numerical values that most patients struggle to interpret. This confusion causes unnecessary anxiety and often leads to unreliable internet searches for answers.

### Community Level
Local health workers (ASHA workers), clinic administrators, and public health departments lack visibility into **aggregate health trends** across the populations they serve. Lab reports exist as unconnected PDFs on individual phones and clinic desks — no one sees the bigger picture. When 30% of patients in a region show elevated blood sugar markers, no alarm goes off. When anemia rates spike among a particular age group, it goes unnoticed until it becomes a crisis.

### What This Project Solves
This platform bridges **both gaps** with a single system:
1. **Patients** upload their reports and get clear, AI-generated explanations grounded in their actual report data.
2. **Health workers & planners** see anonymized, aggregated trends across all uploaded reports — flagging risks, surfacing patterns, and enabling proactive public health response.

### Data Flow

**Patient Mode Flow:**
```
Patient uploads PDF → Extraction Agent → Lab Value Parser → Anomaly Detector
  → Risk Card displayed → Patient asks question → QA Agent retrieves context
  from ChromaDB → LLM generates explanation with source attribution
```

**Community Mode Flow:**
```
Health worker uploads bulk PDFs → Extraction Agent processes each
  → Structured values stored in SQLite (anonymized)
  → Dashboard shows: Top abnormal tests, flag distribution,
    region/age breakdowns, trend charts, community alerts
  → Predictive Forecasting: linear regression projects abnormal rates
    30-90 days ahead per test and per region → risk heatmap
  → Health worker asks NL question → Community Agent queries SQLite
    + forecast data → LLM generates narrative insight with predictions
    and actionable recommendations
```

## 📖 Example Usage

### Patient Mode
```
Patient: "What does my hemoglobin mean?"
→ AI retrieves hemoglobin-related sections from the uploaded report
→ Explains the value, whether it's normal, and what it means
→ Shows source evidence from the report alongside the answer
→ Includes medical disclaimer
```

### Community Mode
```
Health Worker: "Which locality had the most abnormal blood sugar readings this month?"
→ Community Agent queries SQLite for aggregated HbA1c/glucose data
→ Groups by anonymized region, calculates abnormal percentages
→ Returns: "Rural-West shows a 42% abnormal rate for fasting glucose,
   significantly higher than the 18% average across other regions.
   Consider organizing a diabetes screening camp in this area."
```

## 🛡️ Responsible AI

This project implements responsible AI principles throughout:

| Principle | Implementation |
|---|---|
| **Source Attribution** | Every AI answer shows the retrieved report sections used as evidence |
| **Grounded Responses** | RAG ensures answers come from actual report data, not LLM hallucination |
| **Confidence Disclosure** | System explicitly states when it cannot find relevant information |
| **Medical Disclaimer** | Prominent, mandatory disclaimer on every response and in the UI header |
| **Privacy Protection** | System prompt prohibits revealing patient names/IDs; community data is anonymized |
| **Anomaly Transparency** | Flagging uses explicit reference ranges — users can see why a value was flagged |
| **No Independent Diagnosis** | LLM is instructed to summarize findings, never invent new diagnoses |

## 🚀 Installation

### Prerequisites
- Python 3.10+
- A Google Gemini API key

### Setup

```bash
# Clone the repository
git clone https://github.com/Astroidkiller/medical-report-rag.git
cd medical-report-rag

# Install dependencies
pip install -r requirements.txt

# Create .env file with your API key
echo "GEMINI_API_KEY=your_api_key_here" > .env

# Run the application
python -m streamlit run streamlit_app.py
```
